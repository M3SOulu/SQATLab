Instructions:

  • Open eclipse with Besouro TDD plugin. (Download from here if you haven't downloaded last week.
  • Game of life exercise description
    • Check out this online simulator to see how it works.
  • Go to the Github repository of the exercise: Game of life exercise project repository
  • Select branch dev
  • Fork your own copy to your account
  • Go to your repositories page on Github. Link should be: https://github.com/YourGithubaccountName/GameOfLife
  • Select branch dev Link should be: https://github.com/YourGithubaccountName/GameOfLife/tree/dev

Instructions: Set up eclipse project and TDD plugin

  • Import your copy into Eclipse (In Eclipse File > Import > Git > Projects from Git > Clone URI)
    • Import using the clone URL on your Github page (e.g. https://github.com/YourGithubaccountName/GameOfLife.git
    • Choose dev branch to import
  • Open Besouro TDD Plugin: (In Eclipse Window > Show View > Other > Other > episodeView
  • Open a source file and start Besouro TDD Plugin by clicking play button on plugin window. You should see "recording GameOfLife" in the plugin Window.

Instructions: Coding

  • Remember Test-Code-Refacor cycle
  • Each cycle should be as short as possible
  • Try to focus on small tasks
  • Simplest implementation!! Fake it if the implementation is not obvious
  • REFACTOR, REFACTOR, REFACTOR (Extract variable, method) When refactoring do not change behavior

Instructions: Test Selection Strategies

  • Details first vs big picture first
  • happy path vs error situations
  • Try to focus on small tasks
  • high value vs. low hanging fruit
  • exploring the uncertain vs. sticking with the familiar

Instructions: Updating your repository and submitting your work

  • In Eclipse open Git Repositories: Show View > Other > Git > Git Repositoies
  • In Eclipse in Git Repositories Window: right-click your repository and Commit
  • In Eclipse in Git Repositories Window: right-click your repository and Push
  • Go to you repository page on Github.com and check that it is updated.
  • When you are done create a Pull request.
