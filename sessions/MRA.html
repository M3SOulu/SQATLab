Mars Rover API
Use a test-driven development approach to develop an API that moves a Rover around on a planet represented as a grid.

  • The Rover’s movements are based on coordinates (x,y), and directions (N,S,E,W)
  • The Rover receives a character array of commands
    • Move back: b
    • Move forward: f
    • Turn left: l
    • Turn right: r
  • Once a sequence of command is completed the Rover should report its coordinates and direction (a string like “(4,5,N)”)
  • Since the planet is a sphere, the Rover should wrap around the edges of the grid
  • The Rover should detect eventual obstacles before moving to a new square.
    • If a given sequence of commands encounters an obstacle(s), the Rover moves to the last possible square and reports the obstacles (a string like “(4,5,N)(2,4)(1,3)”).
  • Example: The Rover is on a 100x100 planet. It always starts it’s journey at coordinates (0,0) facing North (N). Once the Rover executes the command “ffrff” it reports “(2,2,E)”.
